# Списки доменов для блокировки

Поможем провайдерам в эпоху дефицита сетевого оборудования: самостоятельно заблокируем ресурсы на своих роутерах и, таким образом, снизим нагрузку на их оборудование!

Зарубежные сервисы пусть знают, что их ресурсы никому не нужны и мы сами у себя их блокируем!

Проект основан на [itdoginfo/allow-domains](https://github.com/itdoginfo/allow-domains). В оригинале нельзя распределить сервисы по разным секциям Podkop — здесь это реализовано через Minus-файлы либо вручную собирая группы из непересакающихся доменов.  
Списки сделаны для роутеров под OpenWRT (Podkop/sing-box) и iPhone (Shadowrocket).

## Форматы файлов

| Формат               | Расширение       | Совместимость                            |
| -------------------- | ---------------- | ---------------------------------------- |
| Домены / подсети     | `.lst`           | Podkop, dnsmasq, sing-box, универсальный |
| Surge / Shadowrocket | `-surge.list`    | Shadowrocket (iPhone), Surge, Stash      |
| Sing-box rule-set    | `.srs` + `.json` | sing-box, Hiddify                        |

Рядом с каждым `.lst` лежат те же записи в других форматах: `-surge.list` (Shadowrocket/Surge) и sing-box rule-set `.srs` с исходным `.json`. Surge-формат — это `DOMAIN-SUFFIX,example.com` (домены) или `IP-CIDR,x.x.x.x/y` / `IP-CIDR6,x::/y` (подсети).

## Как устроены списки

Списки состоят из групп доменов и связанных с ними подсетей.

Все домены хранятся в `domains.csv`. Скрипт `convert.py` генерирует из него все `.lst`, `-surge.list` и sing-box `.srs`/`.json` файлы, включая Minus-файлы.

Не все сервисы можно полностью заблокировать по доменам, некоторые обращаются к своим серверам напрямую по фиксированным IP, минуя DNS. Для таких сервисов одних доменов недостаточно: нужно дополнительно блокировать трафик по IP через поле **Remote Subnet Lists** в Podkop. Скрипт `get-subnets.py` получает актуальные IP-диапазоны из официальных источников (RIPE, Cloudflare, Telegram) и создает списки подсетей.

У 6 доменных категорий есть соответствующие списки IP-подсетей; для них предусмотрены и доменные, и subnet Minus-файлы — чтобы при разбивке по секциям Podkop домены и IP-диапазоны оставались согласованными.

Отдельно есть категория **Zscaler** — только IP-подсети, без доменов. Zscaler Client Connector (ZIA-туннель и ZPA) гонит трафик на IP узлов/брокеров, а реальные домены спрятаны внутри TLS/DTLS-туннеля — поэтому ловить его можно только по IP-диапазонам. Для неё есть subnet Minus-файлы.

Отдельно есть категория **Hosters-IP-Blocked** — сайты, которые ограничивают не страну, а сами адреса хостингов и дата-центров. Пример — `adr.it`: CloudFront отвечает `403 Request blocked` с любого арендованного сервера (offshore, Hetzner, RU), но пропускает обычные пользовательские адреса. Через VPN-выход такой сайт не открыть, поэтому список вынесен отдельно: вместе с парным `minus-21-hosters-ip-blocked` он позволяет завести отдельную секцию Podkop, направленную на чистый резидентный не-РФ адрес.

Для категорий, к которым нет списков подсетей, блокировка возможна просто по доменному имени.

## Сравнение списков доменов с upstream

Проект основан на [itdoginfo/allow-domains](https://github.com/itdoginfo/allow-domains). Последняя проверка — коммит `5c285ec` (2026-07-15): новых доменов нет (в апстриме только обновление подсетей). Чтобы посмотреть, какие домены добавили в апстрим с тех пор:

```bash
# Подключить апстрим как второй источник (один раз):
git remote add upstream https://github.com/itdoginfo/allow-domains.git

# Проверка:
git fetch upstream
```

## Sing-box / Hiddify

Списки доступны как sing-box rule-set: `.srs` (для клиента) и `.json` (исходник) — рядом с каждым `.lst`, в тех же папках и с теми же именами. Подключаются в секции `route` конфига sing-box, который исполняет Hiddify (отдельного поля под кастомные rule-set в самом приложении нет — правила задаются в конфиге).

Три шага:

1. **Подключить список** — блок в `rule_set` с raw-URL нужного `.srs`; `tag` — любое имя.
2. **Направить трафик** — в `rules` перечислить эти теги и задать `outbound`: `proxy` (через VPN) или `direct` (напрямую).
3. **Остальное** — `final` (напр. `direct`: весь несовпавший трафик идёт напрямую).

`proxy`/`direct` — теги ваших outbound'ов. У сервисов с фиксированными IP к доменному списку подключайте и подсети из `Subnets/IPv4/` и `Subnets/IPv6/`, иначе IP-трафик обойдёт правило.

```json
{
  "route": {
    "rule_set": [
      {
        "type": "remote",
        "tag": "youtube",
        "format": "binary",
        "update_interval": "3d",
        "url": "https://raw.githubusercontent.com/haritos90/allow-domains/main/Categories/01-youtube.srs",
        "download_detour": "proxy"
      },
      {
        "type": "remote",
        "tag": "telegram",
        "format": "binary",
        "update_interval": "3d",
        "url": "https://raw.githubusercontent.com/haritos90/allow-domains/main/Categories/04-telegram.srs",
        "download_detour": "proxy"
      },
      {
        "type": "remote",
        "tag": "telegram-v4",
        "format": "binary",
        "update_interval": "3d",
        "url": "https://raw.githubusercontent.com/haritos90/allow-domains/main/Subnets/IPv4/04-telegram.srs",
        "download_detour": "proxy"
      },
      {
        "type": "remote",
        "tag": "telegram-v6",
        "format": "binary",
        "update_interval": "3d",
        "url": "https://raw.githubusercontent.com/haritos90/allow-domains/main/Subnets/IPv6/04-telegram.srs",
        "download_detour": "proxy"
      }
    ],
    "rules": [
      {
        "rule_set": ["youtube", "telegram", "telegram-v4", "telegram-v6"],
        "outbound": "proxy"
      }
    ],
    "final": "direct"
  }
}
```

- **Всё сразу через прокси** — `Russia/russia-all.srs` (домены) + `Subnets/IPv4/all.srs` и `Subnets/IPv6/all.srs` (подсети) → `proxy`.
- **Наоборот** (весь трафик уже идёт через VPN) — `Russia/russia-outside.srs` → `direct`.

`.srs` — скомпилированный формат, в текстовом редакторе не читается. Чтобы посмотреть содержимое, открой соседний `.json` (это исходник) или выполни `sing-box rule-set decompile file.srs`.
